- **维度**：服务化
- **级别**：要求

| 项目 | 内容 |
| --- | --- |
| **量化目标** | 一个容器只运行一个进程 |
| **验收方式** | DockerFile和部署模版中指定的启动脚本指启动单一进程 |
| **描述** | 一个容器只运行一个进程。复杂容器的构建时间非常长、镜像体积大、横向扩展时非常浪费资源（不同的应用需要运行的容器数并不相同）、个进程会混合日志输出难以追踪和诊断、进程崩溃可能影响多个进程。一个容器只运行一个进程遵循单一职责原则，确保容器的高内聚、低耦合，从而实现高效的编排、故障隔离、资源管理和可观测性。 |
