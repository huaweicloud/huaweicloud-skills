# 验收标准

> 当需要判定 Skill 开发任务是否完成时读取此文件。

## 验收条件

Skill 必须同时满足以下所有条件才算验收通过：

### 结构验收

| 条件 | 判定标准 | 验证方法 |
|------|----------|----------|
| SKILL.md 存在 | 文件存在且非空 | `test -s SKILL.md` |
| Frontmatter 完整 | name + description 必填 | `validate-skill.sh` |
| 目录结构完整 | references/ 存在 | `test -d references/` |
| 命名规范 | 符合 `{platform}-{product}-{function}` | 正则匹配 |

### 内容验收

| 条件 | 判定标准 | 验证方法 |
|------|----------|----------|
| description 结构化 | 包含 5 点编号描述 | 检查 `1.` `2.` `3.` |
| 概述章节 | 包含背景和定位 | 检查 `## 概述` |
| 主要步骤 | 每步有 CLI 指令 | 检查代码块 |
| 使用示例 | 3-5 个典型示例 | 计数 `### 示例` |
| 权限标注 | 每个操作标注所需权限 | 检查 `所需权限` |

### 安全验收

| 条件 | 判定标准 | 验证方法 |
|------|----------|----------|
| 无硬编码 AK/SK | SKILL.md 和脚本中无明文凭证 | `validate-skill.sh` 安全检查 |
| 环境变量获取 | 敏感信息通过 `${VAR}` 获取 | grep 检查 |
| 写入确认 | 创建/修改/删除操作有确认提示 | 检查正文描述 |
| dry-run | 高危操作有预览模式说明 | 检查 `--dry-run` |

### 权限验收

| 条件 | 判定标准 | 验证方法 |
|------|----------|----------|
| iam-policies.md | 文件存在 | `test -f references/iam-policies.md` |
| 查询/操作分表 | 两类操作分别列出 | 检查章节标题 |
| 最小权限 JSON | 包含策略 JSON | 检查 `json` 代码块 |
| MFA 标注 | 高危操作标注是否需要 MFA | 检查 MFA 列 |

### 文档验收

| 条件 | 判定标准 | 验证方法 |
|------|----------|----------|
| references 引用正确 | 被引用文件全部存在 | `validate-skill.sh` |
| ref 文件专注 | 每个 ref 解决一个问题 | 人工审查 |
| ref 文件有头部说明 | 文件首行说明何时读取 | 人工审查 |
| 代码块标注语言 | 所有代码块有语言标识 | 检查 ` ```bash ` 等 |

## 验收流程

1. 运行 `bash scripts/validate-skill.sh {skill-path}` 自动检查
2. 确认输出 Grade 为 A 或 B
3. 人工审查内容质量（示例准确性、步骤完整性）
4. 如有脚本，测试脚本可执行性
5. 全部通过则验收完成

## 验收评级

| 评级 | 条件 | 后续动作 |
|------|------|----------|
| 通过 | Grade A/B + 人工审查通过 | 可发布 |
| 有条件通过 | Grade B + 部分推荐项未通过 | 可发布，创建改进 Issue |
| 不通过 | Grade C/D 或人工审查未通过 | 需修复后重新验收 |
